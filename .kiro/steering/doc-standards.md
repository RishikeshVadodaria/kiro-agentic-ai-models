# Documentation Standards

## Code Documentation
- Use clear, concise docstrings for all functions, classes, and modules
- Document parameters, return values, and exceptions
- Include usage examples for complex functions
- Keep comments up-to-date with code changes

## API Documentation
- Document all endpoints with clear descriptions
- Include request/response examples
- Specify required vs optional parameters
- Document error responses and status codes

## README Files
- Include project overview and purpose
- Provide clear installation and setup instructions
- Document configuration options
- Include usage examples and common workflows

## Technical Documentation
- Use markdown format for consistency
- Structure with clear headings and sections
- Include diagrams for complex architectures
- Keep documentation close to the code it describes

## Standards
- Write documentation for your future self and teammates
- Use consistent terminology throughout
- Update documentation with every code change
- Review documentation during code reviews
